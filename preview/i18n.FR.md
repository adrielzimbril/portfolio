# 🌍 Internationalisation (i18n)

Le projet utilise `next-intl` pour offrir une expérience multilingue complète (Anglais, Français et Chinois).

## 📁 Emplacements des Fichiers

- **Fichiers JSON de traduction**: `src/integrations/i18n/translations/`
    - `fr.json` (Source/Référence)
    - `en.json`
    - `zh-CN.json`
- **Données personnelles**: `src/data/personal/translate/`
- **Contenu MDX**: Les fichiers localisés utilisent le format `[nomdufichier].[locale].mdx` (ex: `about.en.mdx`) dans `src/content/`.

## 🤖 Traduction Automatisée (Lobe i18n)

L'ensemble du projet est traduit automatiquement à l'aide de **[Lobe i18n](https://github.com/lobehub/lobe-cli-i18n)**.

### Comment Traduire
Pour lancer la traduction automatisée, utilisez la commande suivante :
```bash
pnpm translate
```

### Configuration
Vous pouvez personnaliser le comportement de la traduction dans le fichier **`.i18nrc.ts`** à la racine du projet.

#### Modes de Traduction
Dans `.i18nrc.ts`, vous pouvez basculer entre deux modes en modifiant l'objet `config` :

```typescript
const config = {
  modelName: "gpt-5-nano",
  mode: "default", // "default" pour les fichiers JSON | "content" pour les données personnelles
  isMdx: false,    // Mettre à true pour traduire les fichiers MDX | false pour les fichiers MD standards
};
```

- **`mode: "default"`**: Traduit les chaînes de l'interface principale dans `src/integrations/i18n/translations/`.
- **`mode: "content"`**: Traduit les données dans `src/data/personal/translate/`.
- **`isMdx: true`**: Cible tous les fichiers `.mdx` du répertoire `src/content/`.

## 🛠️ Utilisation dans le Code

### Composants Client
```tsx
import { useTranslations } from 'next-intl';

const t = useTranslations('home');
return <h1>{t('title')}</h1>;
```

### Composants Serveur
```tsx
import { getTranslations } from 'next-intl/server';

const t = await getTranslations('home');
return <h1>{t('title')}</h1>;
```

## ➕ Ajouter de Nouvelles Traductions

1. Ajoutez d'abord la clé et sa valeur dans **`fr.json`**.
2. Lancez `pnpm translate` (en mode `default`) pour générer automatiquement les versions anglaise et chinoise.
3. Utilisez la clé dans votre composant à l'aide du hook `useTranslations` ou `getTranslations`.
